FPGA Doohickey

A Cyclone IV FPGA-powered interactive hardware toy using buttons and potentiometers as inputs and an I2C OLED screen & LEDS as outputs - all coded in SystemVerilog.

What is this?

This is a small, self-contained FPGA toy built to be touched, twisted, and messed with.

Physical inputs: buttons, switches, knobs, and sliders. They drive visible outputs: an OLED displaying a live waveform and a set of LEDs running reactive patterns. There’s no software UI, no microcontroller, and no abstraction layer.

Why build it?

I want a physical FPGA project that wasn’t a lab assignment or a project built purely for a resume. The goal is simple:

  • Build a real FPGA-based system
  • Make something fun enough that kids would actually want to play with (while waiting for class)

For younger students, it’s immediately engaging. Lights animate on their own, patterns shift, and every control produces an obvious response.

For older students, it’s educational without being watered down. Controls map directly to real concepts: amplitude, frequency, signal inversion, function selection, and pattern generation.

Under the hood, it’s all pretty straightforward digital design: synchronous logic, timing, state machines, pseudo-random behavior, and simple waveform generation, all of which implemented directly in hardware.
