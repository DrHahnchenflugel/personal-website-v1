FPGA Doohickey

A Cyclone IV FPGA-powered interactive hardware toy using buttons and potentiometers as inputs and an I2C OLED screen & LEDS as outputs - all coded in SystemVerilog.

What is this?

  • This project is a small, self-contained FPGA "toy" designed to be poked, prodded, twisted, and experimented with.

    • Physical inputs (buttons, switches, knobs, sliders) directly control visible outputs: an OLED transforming a live waveform and a bunch of LEDs running patterns that also react to user input. There's no abstraction layer, no software UI, and no microcontroller doing the thinking - the FPGA is the system itself.
I wanted a physical FPGA project that wasn't a lab assignment, or a project for a project's sake. The goal is twofold. I wanted:
  • An FPGA project
  • A toy that kids could play with while waiting for class

For younger students, it should be physically engaging - lights animate on their own, patterns shift, and turning a knob or pushing a button obviously does something.

For older students, it should be something they can actually learn from. THe controls are mapped to real concepts: amplitude, frequency, signal inversion, function selection, and pattern generation. Also, flashy lights.

Under the hood, it's meant to demonstrate digital design ideas: synchronous logic, timing, statem machines, pseudo-random behavior, and simple waveform generation. It does all this without hiding everything behind a CPU or firmware layer.
