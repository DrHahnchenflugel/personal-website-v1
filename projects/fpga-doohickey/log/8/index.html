The I2C Protocol

  • The master initiates the communication by generating a START condition. (Done, confirmed working)
  • The slave address is following the start condition ... either "b0111100" or "b0111101". (Done, confirmed working)
  • The write mode is established by setting the R/W bit to logic "0". (Done, confirmed working)
  • An acknowledgement signal will be generated after receiving one byte of data, including the slave address and R/W# bit. (Add acknowledge failure handling)
  • After the transmission of the slave address, either the control byte or the data byte may be sent across the SDA. A control byte mainly consistes of Co and D/C# bits following by 6 "0"'s. (Add control/data byte transmission)
  • If the Co bit is set as logic "0", the transmission of the following information will contain data bytes only. (Add control/data byte transmission)
  • The D/C# bit determines the next data byte is acted as a command or a data. If the D/C# bit is set to logic "0", it defines the following data byte (D[7:0]) as a command. If the D/C# bit is set to logic "1", it defines the following data byte (D[7:0]) as a data which will be stored at the GDDRAM. The GDDRAM column address pointer will be increased by one automatically after each data write. (Add D/C# bit handling)
  • Acknowledge bit will be generated after receiving each control byte or data byte (Done, confirmed working)
  • The write mode will be finished when a stop condition is applied. (Add general write structure)

The Graphic Display Data RAM (GDDRAM)

As opposed to... the graphic non-display?
The GDDRAM is a bit-mapped static RAM that holds the bit pattern to be displayed. The size of the RAM is 128x64 bits, equal to the pixel-wise size of the screen, and is divided into 8 pages, from PAGE0 through PAGE7, which are used for monochrome 128x64 dot matrix display.

Fig. 8-13, above, shows the way the pages are structured. Fig. 8-14, below, zooms into PAGE2 (COM16..23), and shows its individual bit-wise representation.

Let's examine Fig. 8-14. There are 128 columns, SEG0..SEG127. Each column is represented by 1 byte of data, with the data's LSB (D0) being stored in the top byte, COM16, and the data's MSB (D7) stored in the bottom byte, COM23. Thus, whenever one data byte is written into the GDDRAM, all the rows of the current column in the current page are filled.
Notably, the PAGEs can be remapped from the black to the green mapping, as shown in Fig. 8-13 for mechanical flexibility.
There are 3 different memory addressing modes in the SSD1306: page addressing mode, horizontal addressing mode, and vertical addressing mode. (We stan the Oxford comma; just ask Nelson Mandela, the Pope, a demigod and ***** collector). The 20h command sets the way of memory addressing into one of the above 3 modes. There, "COL" means the graphic display data RAM column.

[1] PAGE ADDRESSING MODE (A[1:0]=10xb)

In page addressing mode, after the display RAM is read/written, the column address pointer is increased automatically by 1. If the column address pointer reaches column end address, the column address pointer is reset to column start address and page address pointer is not changed. I have to set the new page and column addresses in order to access the next page RAM content. The sequence of movement of the PAGE and column address point for page addressing mode is shown below, in Fig. 10-1.. In the normal display data RAM R/W and page addressing mode, the following steps are required to define the starting RAM access pointer location:
  • Set the page start address of the target display location by command B0h to B7h.
  • Set the lower start column address of pointer by command 00h~0Fh.
  • Set the upper start column address of pointer by command 10h~1Fh.
For example, if the page address is set to B2h, lower column address is 03h and upper column address is 10h, then that means the starting column is SEG3 of PAG2. The RAM access pointer is located as shown below, in Fig. 10-2. The input data byte will be written into RAM position of column 3.

[2] HORIZONTAL ADDRESSING MODE (A[1:0]=00b)

This one's much simpler. In horizontal addressing mode, after the display RAM is read/written, the column address pointer is increased automatically by 1. If the column address pointer reaches column end address, the column address pointer is reset to column start address and page address pointer is increased by 1. The sequence of movement of the page and column address point for the corizontal addressing mode is shown below, in Fig. 10-3. When both column and page address pointers reach the end address, the pointers are reset to column start address and page start address (Dotted line in Fig. 10-3).

[3] VERTICAL ADDRESSING MODE (A[1:0]=01b)

This is essentially the Horizontal addressing mode rotated 90 degrees and flipped. That is, after the display RAM is read/written, the page address pointer is increased automatically by 1. If the page address pointer reaches the page end address, the page address pointer is reset to page start address and column address pointer is increased by 1. The sequence of movement of the page and column address point for vertical addressing mode is shown below, in Fig. 10-4. When both column and page address pointers reach the end address, the pointers are reset to column start address and page start address (Dotted line in Fig. 10-4).
In order to define the RAM access pointer locations in R/W and horizontal/vertical addressing mode, the following steps are required:
  • Set the column start and end addresses of the target display location by command 21h.
  • Set the page start and end addresses of the target display location by command 22h.


Miscellaneous Info

  • To prevent screen tearing, I need to consider frame synchronisation. That is, if the FPGA writes data to the OLED slower than the OLED's framerate (which, I believe, I can change via command), I have to figure something else out. Maybe update slower, maybe something else.
  • The chip is initialised as follows:
  • Display OFF
  • 128x64 display mode
  • Normal sengment and display data column address and row address mapping (SEG0 mapped to 00h, COM0 mapped to 00h)
  • Shift reg data clear in serial iface
  • Display start line is set at display RAM address 0
  • Column address counter is set at 0
  • Normal scan direction of the COM outputs
  • Contrast control register is set at 7Fh
  • Normal display mode, equivalent to A4h command
